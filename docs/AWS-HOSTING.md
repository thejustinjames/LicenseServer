# AWS Hosting Architecture for License Server

## Overview

This document outlines hosting options for the License Server on AWS, including cost-effective and enterprise-grade configurations.

---

## Can This Be Hosted on AWS Amplify?

### Short Answer: **Partially**

AWS Amplify is designed for:
- Static websites / SPAs
- Serverless backends (Lambda-based)

**What Amplify CAN host:**
- The frontend (`public/` folder) - static HTML/CSS/JS

**What Amplify CANNOT directly host:**
- The Express.js backend (long-running server)
- WebSocket connections (if needed)
- Stripe webhooks (need persistent endpoint)

### Recommended Approach

Use **Amplify for frontend** + **separate backend service** (ECS, App Runner, or Lambda).

## CI/CD Pipeline

### GitHub Actions Workflow

```yaml
# .github/workflows/deploy.yml
name: Deploy to AWS

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: us-east-1

      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push Docker image
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          ECR_REPOSITORY: license-server
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          docker tag $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG $ECR_REGISTRY/$ECR_REPOSITORY:latest
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:latest

      # App Runner auto-deploys on ECR push
      # Or trigger ECS deployment:
      - name: Deploy to ECS
        if: false  # Enable for ECS
        run: |
          aws ecs update-service \
            --cluster license-server \
            --service license-server \
            --force-new-deployment
```

## Security Checklist for AWS

- [ ] Enable VPC for all services
- [ ] Use private subnets for database and Redis
- [ ] Enable encryption at rest (RDS, S3, ElastiCache)
- [ ] Enable encryption in transit (SSL/TLS)
- [ ] Use Secrets Manager for all credentials
- [ ] Enable CloudTrail for audit logging
- [ ] Configure WAF rules for API protection
- [ ] Set up VPC Flow Logs
- [ ] Enable GuardDuty for threat detection
- [ ] Use IAM roles (no hardcoded credentials)
- [ ] Enable MFA for AWS console access
- [ ] Configure backup retention policies

## Cost Optimization Tips

1. **Use Reserved Instances** for RDS (1-year = 30% savings)
2. **Use Spot instances** for non-critical workloads
3. **Enable S3 Intelligent Tiering** for packages
4. **Use CloudFront caching** to reduce origin requests
5. **Right-size instances** based on CloudWatch metrics
6. **Use Aurora Serverless v2** for variable DB workloads
7. **Set up billing alerts** in AWS Budgets
